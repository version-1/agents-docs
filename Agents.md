# agents-doc


## Agentのロール

Agent は次の 4 つの役割を意識してください。

### ① Planner（設計・分解役）

責務

要件を小さなタスクに分解

Doneの定義（DoD）を書く

実装順序を決める

絶対にやらせないこと

実装

細かいコードレビュー

### ② Implementer（実装役）

#### 責務

Planner の指示を忠実に実装

自己判断を極力しない

必ずローカルで検証

#### 禁止事項

仕様の拡張

「ついでに改善」

#### アウトプット

コード差分

実行したコマンドと結果

### ③ Verifier（検証・ツッコミ役）

#### 責務

差分だけを読む

壊れそうな点を列挙

足りないテストを指摘

#### 特徴

- 書かない
- 直さない
- 疑う

## ④ Doc（言語化・整備役）

### 責務

README / ADR / コメント整理

「未来の自分」に説明する

### アウトプット

変更点の要約

設計判断の理由
