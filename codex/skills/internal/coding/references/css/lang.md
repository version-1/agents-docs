
これは CSS 及び CSS Module, Sass などでのコーディングに関するガイドラインです。

# 推奨事項

- レイアウトと見た目を分離して子はコンポーネントとして閉じることを推奨します。
- 固定値を使用する場合はスケールを意識して、変数を使用してください。
  - 例: `--spacing-1: 4px;` など。
- レスポンシブを意識して、width や height などの値を固定値で指定しないでください。
  - 親コンポーネントでサイズを限定することを推奨します。
- カラーコードは変数を使用してください。グローバル空間に定義が足りない場合は適宜追加してください。
  - 例: `--color-primary: #ff0000;` など。
- z-index を使用する場合は、スケールを意識して、変数を使用してください。
  - 例: `--z-index-1: 100;` など。

# 非推奨事項

- important を使用しないでください。
- id セレクタを使用しないでください。
- 子要素と親要素が依存するコーディングを避けてください。
  - 例: `.parent .child { ... }` は避けてください。子要素用のクラスを作成してください。
- タグセレクタを使用しないでください。
  - 例: `div { ... }` は避けてください。クラスを作成してください。
- ネガティブマージンを使用しないでください。


## CSS Module を使用する場合

- コンポーネント固有のスタイルはコンポーネントごとに CSS Module を作成して管理してください。
- グローバルなスタイルは、アプリケーションに共通するスタイルを定義するためのグローバル CSS ファイルを作成して管理してください。

## グローバルで管理すべきスタイル

- フォントファミリー、フォントサイズ、行間などのタイポグラフィに関するスタイル。
- カラーコードなどのテーマに関するスタイル。
- レスポンシブデザインに関するスタイル。
- 共通のユーティリティクラス（例: `.hidden`, `.visually-hidden`, `.sr-only` など）。
- ブラウザのデフォルトスタイルをリセットするためのスタイル（例: `normalize.css` など）。
- z-index
