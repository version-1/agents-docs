
# go 言語における テストコードを書く際のポイント

### ロジックを最小限に止める

- テストコード自体が複雑にならないように、ロジックは最小限にする
- 比較における期待値は関数呼び出し結果などよりは固定値を使用する

### テーブル駆動テストを活用する
  
  - 複数の入力と期待される出力をテーブル形式で定義し、ループでテストケースを実行する
  - コードの重複を減らし、テストケースの追加や変更が容易になる


必要に応じてテストケース用の構造体を定義してテーブル駆動テストができるように工夫する。

```go
type testCase struct {
    name     string
    input    int
    expected int
}
```

構造体のフィールドは、 関数や比較用の構造体を定義しても問題ありません。

```go
type testCase struct {
    name     string
    input    int
    assert   func(t *testing.T, got int)
}
```
