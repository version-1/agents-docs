# SKILL: コード簡略化（Code Simplifier）

## 目的

Agent のコードレビュー結果を反映して、コードをリファクタリングするためのスキル。
挙動の変更は行わずにコードの可読性、保守性、効率性を向上させることを目的とする。

## 入力（ユーザーが指定すること）
- コード
  - git diff, gh prd diff, コミットハッシュ などによるコード差分 
  - パス

## 出力（成果物）
- リファクタリングされたコード（変更されたファイルと行）
- 変更した内容の要約（可読性、保守性、効率性、変更後のリスク）

## 手順

1. git diff や gh pr diff などの形式でコード差分を受け取る。 
2. Agents を観点別に起動して、3 つの観点（可読性、保守性、効率性）でコードレビューを行う。
  - `code-review` skill を使用して、可読性の観点でコードをレビューする Agent を起動する。
  - `code-review` skill を使用して、保守性の観点でコードをレビューする Agent を起動する。
  - `code-review` skill を使用して、効率性の観点でコードをレビューする Agent を起動する。
3. コードレビューの結果をもとに、コードをリファクタリングする。
4. リファクタリングされたコードを出力する。変更されたファイルと行を明示することで、ユーーが変更内容を把握しやすくする。

## リファクタリングの技法

- コードの整理: 共通のコードを関数やクラスにまとめることで、コードの再利用性を高める。
  - 不要なコードの削除
  - コードの分割
  - 関数の抽出
  - コードの統合
  - クラス抽出
  - インターフェース抽出 
  - マジックナンバーの定数化
  - フラグ引数の削除
  - 関数の移動
  - 変数のスコープの縮小
  - 早期リターンの導入
  - 条件分岐のポリモーフィズム化
  - デザインパターンの導入
  - 重複コードの統合
  - 値オブジェクトの導入 
  - データのカプセル化
- 命名の改善: 変数名や関数名をより意味のあるものに変更する。
- コメントの追加: コードの意図や動作を説明するコメントを追加する。
- テストの追加: コードの動作を確認するためのテストを追加する。
- アルゴリズムの最適化: より効率的なアルゴリズムを使用することで、コードのパフォーマンスを向上させる。
- リソースの使用の改善: メモリの使用を最適化する、不要なリソースの解放など。
