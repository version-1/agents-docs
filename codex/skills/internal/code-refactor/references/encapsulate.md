# Encapsulate

## 目的

変更されるべきでない内部状態や実装詳細を隠し、利用側に安定した API だけを見せる。

## 使う場面

- 外部から直接変更できる状態が不整合を生みやすい。
- 利用側が内部構造に依存している。
- 初期化、validation、正規化などのルールが散らばっている。

## 代表的な技法

- フィールドを private にする。
- setter / constructor / factory で不変条件を守る。
- 内部 API を module 内に閉じる。
- data structure への直接アクセスを query / command method に置き換える。

## 注意点

- getter / setter を機械的に増やすだけでは隠蔽にならない。
- テストのためだけに公開範囲を広げない。
- 利用側が必要とする操作を API として表現できているか確認する。

## 報告観点

- 隠した状態や実装詳細。
- 守れるようになった不変条件。
- 呼び出し側の変更範囲。
