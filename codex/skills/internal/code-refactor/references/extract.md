# Extract

## 目的

意味のある処理や重複を切り出し、再利用しやすく、読みやすい単位にする。

## 使う場面

- 同じ処理が複数箇所にある。
- 大きな処理の中に名前を付けられるまとまりがある。
- hook、component、function、value object として独立させると意図が明確になる。

## 代表的な技法

- extract function。
- extract component。
- extract hook。
- extract value object。
- 共通 validation や formatting の切り出し。

## 注意点

- 1 回しか使わない抽出でも、読みやすさが明確に上がる場合は有効。
- ただし、切り出した名前が中身の説明にしかなっていない場合は効果が薄い。
- 抽出した関数に多すぎる引数が必要なら、先にデータ構造や責務境界を見直す。

## 報告観点

- 切り出した単位とその責務。
- 重複削減または読みやすさの改善点。
- 抽出により変えていない入力、出力、エラー挙動。
