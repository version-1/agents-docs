# Move

## 目的

コードを責務に合う場所へ移し、依存方向と発見しやすさを改善する。

## 使う場面

- utility、domain、component、repository などの配置が責務と合っていない。
- あるモジュールだけが使う処理が広すぎる場所に置かれている。
- 共通処理が利用側に散らばっている。

## 代表的な技法

- domain logic を domain 層へ移す。
- UI 専用 helper を component 近くへ移す。
- 汎用 utility と業務固有 helper を分ける。
- テストファイルを対象コードの近くへ移す。

## 注意点

- import cycle やレイヤ違反を作らない。
- ファイル移動だけで責務が変わったように見せない。必要なら名前や API も整える。
- 配置規約がある場合は既存パターンに合わせる。

## 報告観点

- どこからどこへ移したか。
- 移動後の依存方向。
- import / export / test path の更新範囲。
