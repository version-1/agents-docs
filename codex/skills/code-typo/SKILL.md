---

# 目的

- 変更されたコードに typo がないことを確認する 
- 入力で与えらた範囲に typo がある場合は、指摘とともに正しいスペルを提示する

## このスキルが提供するもの
- typo やスタイル違反の指摘
- typo の発見や該当箇所へのナビゲーション(ファイル/行)
- 正しいスペルの提示
- 下記内容のスペルをチェックする
  - コード内容(変数名、関数名、クラス名、コメントなど)
  - ファイルパス（ディレクトリ・ファイル名）
  - コミットメッセージ
  - PR 説明・タイトル 

## このスキルが提供しないもの
- パッチや具体的なコード変更
- アーキテクチャや設計、責務に関する評価・指摘

## 絶対ルール
- コードの変更は行わず指摘に止める

# 入力例

- 変更されたコードの diff（ファイル/行）
- ディレクトリ・ファイルへのパス
- PR の diff
- 変更の目的や背景（コミットメッセージや PR 説明）
- commit ハッシュ
- commit 自体に typo がある場合は、指摘とともに正しいスペルを提示する

# 検査方法

既存のスペルチェックツールを使用するのではなく、下記の手順で typo を指摘することを想定しています。

1. 入力でわたされた範囲のファイルの一覧を取得
2. 逐次的にファイルを読み込み、スペルチェックを行う
3. typo が見つかった場合は、ファイル/行とともに指摘と正しいスペルを提示する
4. 最終的に、指摘された箇所のサマリを出力する

# 出力（成果物）
- 指摘一覧（ファイル/行、 正しいスペルの提示）
- 確認箇所のサマリ
