# 目的
- 変更されたコードに typo がないことを確認する 
- 入力で与えらた範囲に typo がある場合は、指摘とともに正しいスペルを提示する

## このスキルが提供するもの
- typo やスタイル違反の指摘
- typo の発見や該当箇所へのナビゲーション(ファイル/行)
- 正しいスペルの提示

## このスキルが提供しないもの
- パッチや具体的なコード変更
- アーキテクチャや設計、責務に関する評価・指摘

## 絶対ルール
- コードの変更は行わず指摘に止める

# 入力例

- 変更されたコードの diff（ファイル/行）
- ディレクトリ・ファイルへのパス
- PR の diff
- 変更の目的や背景（コミットメッセージや PR 説明）
- commit ハッシュ
- commit 自体に typo がある場合は、指摘とともに正しいスペルを提示する


# 出力（成果物）
- 指摘一覧（ファイル/行、 正しいスペルの提示）
- 確認箇所のサマリ
