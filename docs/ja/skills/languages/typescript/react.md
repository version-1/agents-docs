# React コーディングガイドライン

- コンポーネントは小さく、単一の責務を持つように設計する。
- JSX 内でのロジックは最小限に抑え、必要に応じてヘルパー関数を使用する。
- フック（Hooks）はルールに従い、条件付きで呼び出さない。
- 状態管理には `useState` を適切に使用し、必要に応じてコンテキスト（Context）を活用する。
- 不要な再レンダリングを避けるために、`React.memo` や `useCallback`、 `useMemo` を適切に使用する。
- イベントハンドラーはインラインで定義せず、コンポーネント外で定義する。
- TypeScript の型定義を使用して、コンポーネントのプロパティの型を明確にする。
- スタイルは CSS Modules を使用して管理する。
- コンポーネントのライフサイクルを理解し、適切なフックを使用して副作用を管理する。
- テストは `vitest` や `React Testing Library` を使用して、コンポーネントの動作を検証する。
- アクセシビリティ（a11y）を考慮し、適切な ARIA 属性を使用する。
- React の最新バージョンを使用し、新しい機能や最適化を積極的に取り入れる。

# 推奨ライブラリ

- `react-testing-library`: React コンポーネントのテストを支援するライブラリ。
- `tanstack/react-query`: サーバー状態の管理とデータフェッチングを簡素化するライブラリ。
