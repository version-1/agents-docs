# Go 言語 コーディングガイドライン

- any の使用は最小限にする
- 型アサーションや型スイッチを使って具体的な型を扱う
- reflect パッケージの使用は避ける
- エラーハンドリングは明示的に行う
- defer を使ってリソースの解放を確実に行う
- goroutine の使用は必要な場合に限定し、同期にはチャネルを活用する
- パッケージのインポートは必要最低限に抑える
- コードの可読性を重視し、関数や変数名は意味のあるものにする
- コメントはコードの意図や複雑なロジックを説明するために使用する
- 標準ライブラリや信頼できるサードパーティライブラリを活用する
  - サードパーティライブラリをインストールする際は、人間に必ず確認を行う
- テストコードは充実させ、ユニットテストと統合テストを適切に使い


# 推奨ライブラリ

- HTTP クライアント/サーバー: net/http (標準ライブラリ)
- JSON 処理: encoding/json (標準ライブラリ)
- データベース接続: database/sql (標準ライブラリ) + 各種ドライバ
- ロギング: log (標準ライブラリ) または logrus
- 環境変数管理: os (標準ライブラリ)
- テストフレームワーク: testify あるいは go-cmp
- マイグレーション: プロジェクト標準に準じる
- コンテキスト管理: context (標準ライブラリ)
