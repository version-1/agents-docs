
# TypeScript コーディングガイドライン

- 型安全を最大限に活用し、可能な限り明示的な型注釈を使用する。
- 外部システムのインターフェースには、厳密な型定義を提供し、必ず検証用のコードを追加すること
- 非同期コードには `async` / `await` を使用し、Promise チェーンを避ける。
- 例外処理には `try` / `catch` ブロックを使用し、エラーハンドリングを明確に行う。
- インターフェースと型エイリアスを活用して、コードの可読性と再利用性を向上させる。
- 不要な型アサーションを避け、型ガードを使用して型の安全性を確保する。
- ESLint と Prettier を使用してコードスタイルを統一し、自動フォーマットを導入する。
- モジュールのインポートは、相対パスではなく絶対パスを使用し、プロジェクトのルートからのパス解決を行う。
- ジェネリクスを活用して、柔軟で再利用可能なコンポーネントを作成する。
- 不要な `any` 型の使用を避け、可能な限り具体的な型を使用する。
- 型定義ファイル（`.d.ts`）を活用して、外部ライブラリの型情報を提供する。
- 型の互換性を理解し、型の継承と交差型を適切に使用する。
- コメントはコードの意図を説明するために使用し、冗長なコメントは避ける。
- テストコードには型安全を確保し、型エラーが発生しないようにする。
- TypeScript の最新バージョンを使用し、新しい言語機能を積極的に取り入れる。
- Immutable データ構造を優先し、副作用の少ない関数型プログラミングの原則を取り入れる。


# 推奨ライブラリ

- `tsxe`: TypeScript ファイルを直接実行するためのツール。
- `biome`: TypeScript 用の ESLint プラグインと設定。
- `zod`: スキーマベースの型検証ライブラリ。
- `vitest`: TypeScript 向けの高速テストランナー。
- `react`: TypeScript と組み合わせて使用される人気のあるフロントエンドライブラリ。
- `next`: TypeScript サポートを備えた React ベースのフレームワーク。
