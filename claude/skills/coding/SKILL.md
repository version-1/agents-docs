# Coding スキル

このドキュメントは、コードの実装に関する
**思考の枠組み・判断基準**を定義します。

## 基本姿勢（Core Mindset）

- コードは **人間の理解のための言語** である
- コードの価値は「動くこと」ではなく **理解されること** にある
- 変更は必要最小限にとどめる

## コーディングの判断原則

コードを書くときは以下を常に意識する：
- 既存コードの **流儀・構造・命名** を優先する
- 1つの変更で複数の意図を持ち込まない
- このコードは差分だけ見て理解できるか
- 将来の自分が意図を推測せずに済むか

## コーディングのベストプラクティス

- テスト可能なコードを書く
- 関心の分離を保つ
- 状態とロジックを分離する
- 可読性と保守性を重視する
- 条件分岐を、クラス構造やインターフェースに置き換えて、責務ごとに振る舞いを分離する
- 関数を小さく保ち、テスト可能な範囲に分割する。
- コントラクト層（API/型）を厳密に定義し、実装層は再生成可能に保つ
- 意味のある名前をつける
- コメントは「なぜ」を説明するために使う

言語やフレームワークのベストプラクティスも常に意識すること。
